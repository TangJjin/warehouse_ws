# warehouse_ws

## 项目简介

`warehouse_ws` 是一个基于 ROS 2 Humble 的无人机工作区，用于地瓜派机载电脑上的智能仓储场景。

该项目围绕仓储环境中的自主飞行、定位、任务执行、机载感知与系统联动展开，形成了从飞控通信、激光雷达定位、任务控制到感知识别的完整链路。

## 典型应用场景

- 智能仓储环境中的无人机自主巡检
- 基于预设任务的自动起飞、航线执行与动作控制
- 结合激光雷达建图/定位的稳定飞行
- 结合机载视觉的目标识别、条码识别或图像回传

## 系统特点

- 基于 ROS 2 多包工作区组织
- 机载电脑默认面向地瓜派部署
- 采用 `mavros` 与飞控通信
- 采用 Livox MID360 与 FastLIO 进行定位相关处理
- 支持任务配置、执行控制、状态联动与感知数据接入

## 主要功能链路

典型链路如下：

1. `mavros` 与飞控建立通信
2. `livox_ros_driver2` 接入 MID360 雷达数据
3. `fast_lio` 输出里程计/定位结果
4. `drone_localization` 将定位结果桥接到 MAVROS
5. `drone_control` 建立控制所需 TF 与任务执行链路
6. `drone_perception` 处理机载感知输入
7. `drone_bringup` 负责整体启动编排与运行监督

## 工作区包说明

### 1. drone_bringup
用于系统启动编排、节点拉起、状态等待与启动链监督。

主要作用：
- 按顺序拉起核心节点
- 等待关键 topic 就绪
- 管理机载启动流程
- 配合自启动 service 完成开机自动运行

### 2. drone_control
用于任务控制、动作执行以及飞行控制相关的桥接逻辑。

主要作用：
- 建立飞行控制所需 TF
- 执行任务流程中的动作控制
- 对接 mission 配置与实际飞行执行
- 为 offboard 控制链路提供支撑

### 3. drone_localization
用于定位结果桥接与定位一致性监测。

主要作用：
- 将 FastLIO 输出转换并桥接到 MAVROS
- 向飞控提供视觉/外部定位输入
- 对视觉位姿与飞控本地位姿进行偏差监测
- 为异常检测与重启判据提供依据

### 4. drone_mission
用于任务配置管理。

主要作用：
- 存放任务 YAML 配置
- 描述航点、动作或流程参数
- 作为任务执行时的配置输入

### 5. drone_msgs
用于定义工作区公共消息与服务接口。

主要作用：
- 提供各功能包之间共享的 msg/srv
- 统一任务、状态与感知相关的数据接口
- 降低跨包通信耦合

### 6. drone_perception
用于机载感知接入与识别结果处理。

主要作用：
- 接入 K230 相机侧的 TCP/JPEG 数据
- 处理图像与感知结果
- 支持目标识别、条码相关能力与图像链路对接
- 为仓储识别类任务提供输入

### 7. drone_qt_2
用于机载端或中转侧的业务支撑程序。

主要作用：
- 承接部分任务数据落地或转发
- 参与地面端与机载端之间的数据衔接
- 配合任务启动与感知结果回传链路

### 8. drone_warehouse
用于仓储场景相关功能的预留或专项组织。

目前从包信息看，该包在工作区中已存在，但公开描述较少。按命名和项目用途判断，它主要面向智能仓储场景的业务扩展。

## 依赖组件

项目的重要外部依赖包括：

- `mavros`
- `livox_ros_driver2`
- `fast_lio`

根据具体部署，还可能依赖：

- ROS 2 Humble
- Livox MID360
- 飞控系统与对应 MAVLink 通信链路
- 机载视觉硬件（如 K230）
